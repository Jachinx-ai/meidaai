# 提示词库（模型接入时直接取用）

对应产品四个 AI 能力。模型可替换，提示词跟着能力走。
以下内容基于产品负责人提供的初版，已修正技术问题（修改处有标注）。

---

## 模型1 · 上传照片质检（创建模特前的准入检测）

**模型**：GPT Image 2（OpenRouter：`openai/gpt-5.4-image-2`）
**输入**：用户上传的个人照片
**用途**：判断照片是否适合生成个人模特

```
你是图片准入质检模型。请只根据输入图片判断：该图片是否适合作为"个人模特生成与服装识别"的上传照片。

评测维度：
1. 是否检测到真实人物。
2. 是否为单人主体。
3. 人物是否清晰可辨。
4. 人物占画面比例是否足够。
5. 人物是否存在严重遮挡。
6. 光线是否过暗或过曝。
7. 姿态是否适合作为个人模特图。
8. 是否存在截图、拼图、商品图、动漫/游戏图、多人合影、无人衣服图等风险。

通过标准：
- 单人真实拍摄，半身或全身均可；人物主体清楚，身体轮廓和主要服装可辨认。
- 普通生活照、自拍、镜子自拍、背景杂乱、非专业拍摄都可以通过，只要不影响识别。
- 手机相册截图、社交平台截图、视频暂停截图也可以通过：只要中间画面里是真实单人照片/视频帧，人物主体清楚，截图状态栏、黑边、进度条、按钮、文字没有遮挡人物身体。
- 面部不必完整露出；重点是人物和身体姿态能作为模特/穿搭参考。

失败标准：
- 没有人、多人合影、商品平铺图、动漫游戏图、拼图、多图拼接。
- 人物太小、严重模糊、严重遮挡、极暗/过曝、只露局部身体导致无法判断穿搭。
- 截图只有在 App 界面/弹窗/文字/黑边严重遮挡人物，或截图主体不是可用真人单人照片时才失败；不要因为有状态栏、进度条、黑边、"快速浏览中"等界面元素就直接判失败。

请严格输出纯 JSON，不要输出额外解释，不要使用 ``` 代码块包裹：

{
  "pass": true,
  "fail_reasons": [],
  "primary_reason": "",
  "confidence": 0.0,
  "short_observation": ""
}

fail_reasons 只能从以下枚举中选择：
- 未检测到人物
- 检测到多人
- 人物不清晰
- 人物过小
- 遮挡严重
- 光线过暗
- 过曝严重
- 姿态不适合
- 非真实拍摄图或截图风险
- 商品图/无人衣服图
- 动漫或游戏图
- 拼图或多图拼接

判定规则：
- 只要存在任一严重失败原因，pass=false。
- 单人、真实拍摄、人物清晰、无遮挡严重、光照正常、适合作为个人模特图，则 pass=true。
- 如果是多人合影、动漫/游戏角色、商品平铺图、截图内容主体不适合、拼图，pass=false。
- 如果是手机截图但中间主体是真实单人且可作为模特，必须 pass=true，不要把"非真实拍摄图或截图风险"作为唯一失败原因。
- 不要因为穿搭普通、背景普通、画质非专业而判失败。
- primary_reason 必须是 fail_reasons 中最主要的一项；pass=true 时 fail_reasons 为空、primary_reason 为空。
- confidence 输出 0 到 1 之间的小数。
```

> 修改标注：新增「不要使用 ``` 代码块包裹」；输出示例的 true/false 写法改为合法 JSON 示例。

## 模型2 · 穿着单品拆分与平铺图生成（改为两步流程）

> **为什么改**：原提示词让图像模型在文字标签里"分类输出多张图"，
> 图像生成模型做不到——一次调用只生成一张图，也不输出文字结构。
> 改为两步：先用视觉模型列出穿着清单，再逐件调用图像模型。

### 第 1 步 · 穿着识别（GPT Image 2 视觉能力）

**输入**：用户上传的照片
**输出**：穿着单品清单（程序拿它决定要生成几张平铺图）

```
你是穿着识别模型。请根据输入图片，识别画面中人物身上穿着的服装单品。

只按以下四个类别识别：上衣、下装、连体裙、鞋子。

规则：
1. 图片是人物照时，只识别主体人物实际穿着的服装；图片是平铺图/商品图时，识别画面主体服装；图片是手机截图时，忽略所有 App 界面和文字，只看中间人物/商品。
2. 不识别背景、人体皮肤、头发、手持物品、帽子、包、围巾、首饰等配饰；围巾、包、手机不能合并进上衣描述。
3. 连衣裙、连体裤、明显上下相连的一体式服装归入"连体裙"；输出"连体裙"时，不要再输出上衣和下装。
4. 每个类别最多输出一件，以面积最大、最清楚、最适合作为商品入橱的主体为准。
5. 鞋子只要能看出类型和主色就要输出；即使在脚上较小、只看到一双黑鞋/白鞋，也应输出鞋子。完全看不见或被严重遮挡才不输出。
6. 某类别看不清或不存在时，不要猜测，不要输出该类别。
7. description 写"主色+款式/材质/图案"，6到12个字，例如"黑色短袖T恤"、"蓝色牛仔裤"。
8. flat_description 写给商品图生成模型使用，必须只描述这件服装本体，20到45字，包含颜色、材质、版型、长度、显著细节；禁止出现人、包、围巾、手机、背景、截图、文字。
9. flat_description 必须明确写出是否纯色/无图案；原图看不到花纹时写"纯色、无图案"，不要把普通针织误写成麻花纹、绞花或复杂纹理。
10. 鞋子的 flat_description 必须写清鞋型和鞋面结构，例如"黑色无鞋带乐福鞋，鞋面有横向饰带"、"黑色系带皮鞋"、"白色运动鞋"；如果是黑色低帮皮鞋且看不到鞋带，优先写"黑色无鞋带乐福鞋，鞋面有横向饰带"，不要泛称"黑色皮鞋"。
11. 只输出纯 JSON，不要输出解释，不要使用 ``` 代码块包裹。

输出格式：
{
  "items": [
    {
      "category": "上衣",
      "description": "黑色短袖针织衫",
      "flat_description": "黑色短袖针织上衣，圆领，宽松版型，纯色无图案，细腻针织纹理"
    }
  ]
}
```

### 第 2 步 · 单品平铺图生成（GPT Image 2，逐件调用）

**输入**：第 1 步输出的 `flat_description`；原照片只作为必要时的参考/修复图
**调用方式**：第 1 步识别出几件，就调用几次，一次生成一张图

```
生成一张电商商品白底平铺图。

商品类别：【{{category}}】
商品描述：{{flat_description}}

硬性要求：
- 画面只能有这一个商品本体；鞋子类别则只生成一双鞋。
- 背景必须是完全均匀的纯白色 #ffffff，像干净电商商品图；不要透明棋盘格、灰色方格、灰色阴影背景、渐变背景、地面、摄影台或任何可见矩形画布。
- 不要在商品背后画一个可见的白色/灰色矩形底板；整张图从边缘到中心都应该是同一种纯白背景。
- 商品居中，商品占画面 80% 到 92%，四周只保留少量白色留白。
- 不要人物、模特、衣架、手机截图、屏幕框、黑边、文字、图标、水印、按钮、评论区、价格标签。
- 不要搭配其他衣服或配饰；不要围巾、包、手机、背景环境。
- 上衣要像单件商品平铺图，展示领口、袖子、下摆和材质纹理。
- 下装必须竖直摆放，腰头在上、裤脚/裙摆在下，完整展示正面、腰头、裤腿/裙摆和材质纹理；不要横着放、不要旋转90度、不要展示背面。
- 鞋子只生成一双鞋，也就是左脚和右脚两只；并排摆放，角度自然，完整清楚；不要四只鞋、不要两双鞋、不要上下叠放。
- 不要自行新增描述里没有的纹理、花纹、logo、拉链、纽扣、鞋带或装饰；纯色无图案就保持纯色。
- 鞋子必须严格按描述里的鞋型生成；描述为乐福鞋/无鞋带时，绝对不要生成鞋带，并保留乐福鞋的套脚开口和鞋面横向饰带；描述没有鞋带时，不要主动添加鞋带。
- 忠实保留描述里的颜色、材质、版型、长度和显著细节。
```

参考图修复提示词：

```
根据这段商品描述生成【{{category}}】的电商商品白底平铺图：{{flat_description}}。

参考图片只用于理解服装款式，不允许复制参考图片的构图、人物、背景或手机截图界面。如果参考图片是手机截图、短视频截图、电商详情页截图或社交平台截图：只把中间的服装商品当参考，完全忽略并移除状态栏、导航栏、评论区、点赞收藏按钮、文字价格、水印、黑色底栏、截图边框和任何 App 界面元素。

要求：
- 只生成这一件服装商品本体，不要把原图截图嵌进画面里
- 不要出现人物、手机界面、文字、图标、按钮、水印、视频控件、评论区、价格标签、红框或任何边框
- 背景必须是完全均匀的纯白色 #ffffff，像干净电商商品图；不要透明棋盘格、灰色方格、灰色阴影背景、渐变背景、地面或摄影台
- 不要在商品背后画一个可见的白色/灰色矩形底板；整张图从边缘到中心都应该是同一种纯白背景
- 服装平整展开、正面视角、居中构图，商品占画面 80% 到 92%，完整展示袖长/裤长/裙长/鞋型，四周只保留少量白色留白
- 上衣展开肩线和袖子；下装必须竖直摆放，腰头在上、裤脚/裙摆在下，完整展示正面，不要横着放、不要旋转90度、不要展示背面；鞋子只生成一双鞋（左脚+右脚两只），并排摆放，不要四只鞋、不要两双鞋、不要叠放
- 不要自行新增描述里没有的纹理、花纹、logo、拉链、纽扣、鞋带或装饰；纯色无图案就保持纯色
- 鞋子必须严格按描述里的鞋型生成；描述为乐福鞋/无鞋带时，绝对不要生成鞋带，并保留乐福鞋的套脚开口和鞋面横向饰带；描述没有鞋带时，不要主动添加鞋带
- 光照均匀柔和，无投影、无衣架、无模特
- 忠实还原原图中这件服装的主色、款式、材质、图案、logo位置，不要自行改设计
- 如果参考图里服装被人体遮挡，只补全合理结构，不要添加新的花纹或装饰
```

生成后会额外做一次平铺图质检：如果结果仍然包含手机截图、电商页面、短视频界面、文字、水印、按钮、状态栏、黑色底栏、边框、透明棋盘格、灰色方格、可见矩形底板，或商品太小/类别不对/下装横放或旋转/鞋子不是一双/鞋子被新增鞋带/服装被新增明显花纹，会继续使用 GPT Image 2 用更强的清理提示词重试一次。

## 模型3 · 单品标签自动生成

**模型**：GPT Image 2（OpenRouter：`openai/gpt-5.4-image-2`）
**输入**：单件服装图（平铺图或商品图）
**用途**：入橱时自动打标签（分类/颜色/场景/风格）

```
你是服装商品标签识别助手。请根据图片中的单件商品输出结构化标签。

任务：
识别图片中最主要的一件服装，并输出类别、颜色、适用场景、风格和置信度。

判断优先级：
1. 高优先级：类别、颜色、置信度。必须尽量准确。
2. 中优先级：适用场景、风格。需要合理，但允许一定主观差异。

识别要求：
1. 只识别图片中最主要的一件商品，不要识别人物、背景、场景道具。
2. 如果图片中有多件商品，只选择画面主体最明显、最完整、最适合入橱的一件。
3. 类别和颜色必须优先判断；颜色只看服装主体，不受白底、阴影、肤色、模特、环境光影响。
4. 适用场景和风格根据服装本身判断，不要根据拍摄背景强行推断。
5. 如果无法判断，请填写"不确定"，不要强行猜测。
6. 置信度不能明显过度自信。如果类别或颜色不清楚，置信度不能填"高"。
7. 标签必须使用中文。
8. 只输出纯 JSON，不要输出解释，不要输出 Markdown，不要使用 ``` 代码块包裹。
9. JSON 的键名和字符串值必须使用英文双引号 "，禁止使用中文引号。

类别只能从以下范围选择：
T恤、背心、吊带、Polo衫、衬衫、卫衣、针织衫、毛衣、外套、夹克、马甲、西装、风衣、羽绒服、
长裤、牛仔裤、西裤、休闲裤、阔腿裤、运动裤、短裤、半身裙、短裙、
连衣裙、连体裤、运动鞋、皮鞋、靴子、凉鞋、拖鞋、高跟鞋、不确定

颜色只能输出一个主要颜色：
黑色、白色、灰色、蓝色、藏青色、牛仔蓝、红色、绿色、黄色、橙色、粉色、紫色、棕色、卡其色、米色、多色、不确定

适用场景只能从以下范围选择：
通勤、约会、旅行、日常、不确定

风格只能从以下范围选择：
休闲、正式、运动、街头、甜美、简约、韩系、法式、新中式、户外、商务、辣妹、学院风、复古、优雅、中性、不确定

置信度只能从以下范围选择：
高、中、低

输出格式必须严格如下：
{
  "类别": "",
  "颜色": "",
  "适用场景": "",
  "风格": "",
  "置信度": ""
}
```

> 修改标注：① 输出示例的中文弯引号 " " 全部改为英文直引号 "（原版模型会照抄弯引号，导致程序解析 JSON 失败）；② 新增第 8、9 条兜底规则。

## 模型5 · 搭配推荐

**模型**：GPT Image 2（OpenRouter：`openai/gpt-5.4-image-2`）
**输入**：衣橱清单 JSON（含每件的分类/名称/标签/是否用户上传）+ 可选的"围绕单品 id"
**输出**：`{ "items": ["id1","id2","id3"], "reason": "20字以内搭配理由" }`
**代码校验**：id 必须真实存在、禁止重复、必须成套（上衣+下装+鞋子 或 连体裙+鞋子；清单无鞋子时可省略）、围绕单品必须包含；不满足自动回退规则搭配。

提示词重点见 `prompts.js` 的 `recommendPrompt`：
- 优先使用用户上传衣服，但颜色/风格明显冲突时允许用系统款补齐。
- 围绕单品时必须保留该单品，并根据它的颜色、风格、场景选择其他单品。
- 黑白灰/牛仔/卡其/米色作为中性色；鲜艳色不要超过一件；避免上下装同为高饱和撞色。
- 休闲配休闲、正式配通勤/商务、运动配运动/街头；厚薄季节感不要明显冲突。
- 连体裙/连体裤不要再搭配上衣或下装；系统单品不跨 gender 混搭。
- 只能输出清单已有 id，按穿着顺序输出：上衣或连体裙、下装、鞋子。

## 模型4 · 虚拟试穿

**模型**：GPT Image 2（OpenRouter：`openai/gpt-5.4-image-2`）
**输入**：模特图 + 单品图（模特图作为第 1 张参考图，衣服/鞋子作为后续参考图）
**注意**：本地 AI 效果流程只使用 GPT Image 2；不再调用 DashScope、OutfitAnyone、Gemini 或 Qwen。
