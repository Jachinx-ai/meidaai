# 衣服素材命名规范（给整理素材的 AI/同学）

目标：把收集的穿搭素材图统一命名后，平铺放入项目的 `assets/real/` 文件夹。
命名编码了 性别 / 场景 / 套数 / 单品类别 四层信息，产品数据将据此自动生成。

## 命名格式

```
单品图： <性别>-<场景>-<两位套数>-<类别>.png
整套图： <性别>-<场景>-<两位套数>.png          （可选，一套穿搭的整体展示图）
```

扩展名保持素材原格式（png / jpg 都行，不要做格式转换，PNG 可能带透明底，转 jpg 会毁图）。

### 代码表

| 维度 | 取值 |
|---|---|
| 性别 | `f` = 女生，`m` = 男生 |
| 场景 | `daily` = 日常，`work` = 通勤，`date` = 约会，`travel` = 旅行 |
| 套数 | 两位数字，对应原文件夹序号，有几套编几套（如 `01` ~ `11`） |
| 类别 | `top` = 上衣，`bottom` = 下装，`shoes` = 鞋子，`dress` = 连体裙（连衣裙/连体裤） |

### 示例

女生-通勤场景-第3套 的三件单品：

```
assets/real/f-work-03-top.jpg
assets/real/f-work-03-bottom.jpg
assets/real/f-work-03-shoes.jpg
```

男生-日常场景-第10套（如果有整套展示图）：

```
assets/real/m-daily-10-top.jpg
assets/real/m-daily-10-bottom.jpg
assets/real/m-daily-10-shoes.jpg
assets/real/m-daily-10.jpg          ← 整套图（可选）
```

## 执行要求

1. **复制，不要移动/删除原素材**，原文件夹保持原样
2. 保持原图格式不转换（png 保持 png），文件名一律小写
3. 每个穿搭文件夹里的图片按内容判断类别（上衣→top、裤子/半身裙→bottom、鞋→shoes、连衣裙→dress）；
   连衣裙套装没有 top/bottom，只有 dress + shoes
4. 一套里同类别出现多张时，只取最清晰的一张
5. 完成后输出一份清单：每套穿搭包含哪些类别文件，**并单独列出缺类别的套**（比如某套没有鞋子图）
6. 不要修改项目里的任何代码文件，只往 `assets/real/` 放图片

## 完成后

回到 Claude Code 说一声「素材已就位」，由它扫描 `assets/real/` 生成产品数据
（js/data.js 的衣橱/搭配/场景，含性别筛选），素材不用再动。
