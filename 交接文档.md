## 一、这是什么项目

移动端网页版「AI穿搭助手」（对标 wearwow，黑白极简风），**已经是能跑的真产品，不是原型**：

- 11 个页面全部真实可用：登录 → 引导问卷 → 首页 / 衣橱 / 试穿 / 试衣间 / 我的 等
- 5 个 AI 能力已接真实模型并实测通过：照片质检、衣服识别打标签、白底平铺图（抠图）、
  虚拟试穿（真上身效果）、搭配推荐
- 素材库 230 件单品 / 80 套搭配；数据存浏览器 localStorage，无数据库无登录鉴权

## 二、拿到文件后怎么跑起来（3 步）

```bash
cd server
npm install     # 第一次执行一次
npm start       # → 浏览器打开 http://localhost:8394/login.html
```

- **密钥**：AI 能力需要 `server/.env` 文件（**单独找项目负责人要，不要传到任何公开仓库**）。
  照着 `server/.env.example` 填。没有密钥也能跑：AI 功能自动降级为本地模拟/叠图，适合纯改 UI。
- 手机体验：电脑手机连同一 Wi-Fi，手机浏览器访问 `电脑IP:8394/login.html`
  （查 IP：Mac 终端输 `ipconfig getifaddr en0`）
- 重置演示数据：任意页面 URL 加 `?reset=1`，或「我的」页右上角 ⚙

## 六、开发红线（违反会被打回）

1. 不引入任何框架、打包工具、CSS 框架、新 npm 依赖
2. 不手改 `js/data.js`；不改衣物四分类（上衣/下装/鞋子/连体裙）
3. 不自造颜色：唯一的彩色渐变 `--grad` 只准用在 AI 能力元素上
4. 不删 CLAUDE.md 里标了「勿删」的机制（防烧钱、renderStatusbar 空函数等）
5. 图片一律经 `compressImage()` 压缩再存 localStorage；单品图用 `itemImg(item)` 不手写 `<img>`
6. 素材文件名一律小写（服务器区分大小写，`.PNG` 会 404）

## 八、常见坑（前人踩过的）

- **后端"假离线"**：前端打开页面时探测后端只等 1.5 秒，超时就整页静默降级成本地模拟
  （上传的衣服不抠图、试穿不生成）。遇到"AI 好像没生效"先确认 `npm start` 还活着、刷新页面。
- 试穿模型只支持上装+下装（连体裙算上装），**鞋子不参与生成**，前端是把鞋子叠显在生成图上的
- 试穿页舞台底色是自动取模特图边缘颜色的（`syncStageBg`），不要写死颜色
- 手机上"滑不动/被裁掉"类 bug，优先查 `overflow` / `min-height:0` / `overscroll-behavior`，别加固定高度硬顶
- localStorage 有 5MB 上限，所有进 Store 的图片必须先压缩
