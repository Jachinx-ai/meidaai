# AI穿搭助手 · 高保真可交互原型

这是我们产品的**高保真 UI 原型**：用来演示产品有哪些功能、流程怎么走。
不包含真实功能开发（没有真的 AI、没有服务器），所有交互都是模拟的。

## 怎么看演示

1. 双击打开 **index.html** —— 总览页，11 个页面装在手机壳里一字排开，点任意手机进入交互
2. 建议演示顺序：**登录 → 新用户引导 → 首页 → 试穿 → 添加衣服 → 衣橱 → 试衣历史 → 我的**
3. 用手机看：让电脑和手机连同一个 Wi-Fi，在项目文件夹运行 `npx http-server -p 8394`，手机浏览器访问 `电脑IP:8394/login.html`

## 能真实交互的功能

- 登录/注册全流程（一键登录、邮箱验证码倒计时、协议勾选拦截）
- 新用户引导 4 步问卷 + 挑选单品建衣橱
- 首页按场景筛选搭配、收藏（会保存）、点卡片看穿搭详情
- 模特试穿：底部抽屉换整套/换单件、点身上的衣服弹「替换」、AI 搭配按钮现组一套
- 添加衣服：真的可以从相册选图 → 扫描动画 → 识别 → 进衣橱
- 创建模特：「打开相册」上传成功路径、「现在拍一张」演示失败弹窗
- 试衣历史的编辑删除、我的页收藏/模特管理

数据存在浏览器里（localStorage），刷新不丢；换浏览器/清缓存会重置。

## 想改这个原型？对你的 AI 助手（Claude Code / Codex）这样说

项目里有一份 `CLAUDE.md`，AI 会自动读懂项目规矩。你只需要用大白话下指令，例如：

- 「把首页的天气改成上海 20~26°C 晴」
- 「衣橱页加一个"配饰"分类，里面放两条项链」
- 「参考这张设计图（把图拖进对话），把试衣历史页改成三列瀑布流」
- 「新增一个"会员开通"页面，风格和其他页面保持一致，从我的页进入」
- 「改完帮我 git 存档一次」

改完记得让 AI「**git 提交存档**」，这样随时能回到之前的版本。

## 文件结构（给好奇的人）

```
index.html / login.html / home.html ...   11 个页面，每页一个文件
css/base.css      全局样式（颜色、按钮、卡片等设计系统）
js/app.js         公共逻辑（底部导航、弹层、状态存储）
js/data.js        演示数据（单品、搭配）
assets/ph/        衣物占位剪影图
assets/real/      放真实衣物图片的地方（见里面的 README，放图自动替换）
design-reference/ 对标产品 wearwow 的截图和风格笔记
figma-exports/    Figma 原型图导出
```
